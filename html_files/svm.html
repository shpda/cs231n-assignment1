Inline Question 1:¶

It is possible that once in a while a dimension in the gradcheck will not match exactly. What could such a discrepancy be caused by? Is it a reason for concern? What is a simple example in one dimension where a gradient check could fail? How would change the margin affect of the frequency of this happening? Hint: the SVM loss function is not strictly speaking differentiable

Your Answer:

Yes, it is possible. As suggested by the hint, at the hinge of the loss function (e.g. x = 0 for f = max(x, 0)), the gradient is not defined. This could make the calculated analytic gradient to be quite different from the numerically estimated gradients.

For example, at point x = 1e-9 of function f = max(x, 0). The analytic gradient will be 1, while the numerical gradient could be 0 if -1e-9 is choosed as the delta (i.e. (1e-9 - 1e-9) / (-1e-9) = 0)

Changing the margin can decrease the frequency of this happening if it makes (s_j - s_yi + margin) to be less possible to get a value close to 0.

Inline question 2:¶

Describe what your visualized SVM weights look like, and offer a brief explanation for why they look they way that they do.

Your answer:

The visualized SVM weights still look like the original stuff, just more blurred. For example, one can still roughly see the structure of a car from the car classifier weights.

The reason is that the SVM classifier is doing a simple matrix multiplication, which acts as averaging accross all the pictures in the same class. This ends up making the learned weights to be similar to the original set of pictures.
